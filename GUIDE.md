## 📋 الخطوات

### 1. الحصول على Facebook Cookies

#### الطريقة 1 (من Chrome):
1. افتح facebook.com
2. اضغط `F12` أو `Ctrl+Shift+I`
3. اذهب إلى تبويب `Application`
4. من القائمة الجانبية اختر `Cookies` → `https://www.facebook.com`
5. انسخ جميع الكوكيز (انقر right-click على أي كوكي واختر `Copy all as HTTP header format`)

#### الطريقة 2 (من DevTools):
1. افتح أي طلب (Request) في Network tab
2. انسخ قيمة `Cookie` من Request Headers

**الكوكيز الجيدة يجب أن تحتوي على على الأقل:**
- `c_user` = معرف المستخدم (مثل: 1000123456789)
- `xs` = Session token
- `fr` أو `datr` أو `sb`

---

### 2. الحصول على Ad Account ID

#### من Facebook Ads Manager:
1. افتح https://www.facebook.com/ads/manager/account_settings
2. ابحث عن "Account ID"
3. انسخ الرقم (مثل: `123456789`)

**ملاحظة:** يمكنك إدخال الرقم فقط أو مع `act_` (مثل: `act_123456789`)

---

### 3. الحصول على fb_dtsg (اختياري)

#### أهمية fb_dtsg:
- هذا هو Token الأمني الذي يستخدمه Facebook
- تحتاجه فقط إذا فشل الاستخراج التلقائي

#### الحصول عليه:
1. افتح https://www.facebook.com/
2. اضغط `F12` → `Application` → `Cookies` → `https://www.facebook.com`
3. ابحث عن كوكي باسم `fb_dtsg`
4. انسخ قيمته (عادة تبدأ بـ `1:`)

أو من DevTools Console:
```javascript
document.querySelector('input[name="fb_dtsg"]')?.value
```

## 🚀 استخدام الأداة

### الخطوة 1: إدخال البيانات
```
1. الصق الكوكيز في حقل "ملفات تعريف الارتباط (Cookies)"
2. أدخل معرف الحساب الإعلاني في حقل "Ad Account ID"
3. (اختياري) إذا ظهر خطأ fb_dtsg، أدخله يدوياً
```

### الخطوة 2: بدء الربط
```
اضغط زر "PayPal • بدء الربط"
```

### الخطوة 3: الموافقة في PayPal
```
1. سيفتح نافذة PayPal
2. أدخل بيانات حساب PayPal
3. وافق على الربط
4. ستنقل النافذة تلقائياً البيانات
```

### الخطوة 4: الحصول على البيانات
```
ستظهر البيانات في واجهة المستخدم:
- Token (باللون الأخضر)
- Payer ID (باللون الأزرق)
```

## 🐛 المشاكل والحلول

### خطأ: "لم يتم العثور على fb_dtsg"

**الحل 1:** أدخل fb_dtsg يدوياً في الحقل المخصص

**الحل 2:** تأكد من أن الكوكيز تحتوي على:
- `c_user`
- `xs`
- `fr` أو `datr`

**الحل 3:** حاول تجديد الكوكيز من المتصفح (logout → login)

---

### خطأ: "الكوكيز غير صالحة (لا يوجد c_user)"

**الحل:**
- تأكد من نسخ كل الكوكيز وليس جزء منها
- تأكد من أنك logged in في Facebook
- جرب تجديد الكوكيز

---

### خطأ: "لم يتم إتمام الموافقة"

**الحل:**
- تأكد من إلغاء حظر النوافذ المنبثقة (Pop-ups) في المتصفح
- اتبع الموافقة في نافذة PayPal
- انتظر يتم تحويلك تلقائياً

---

### لا تظهر البيانات في الواجهة

**الحل 1:** تحقق من السيرفر Logs:
```
سترى الرسالة:
========================================
🎉 PAYPAL CALLBACK DATA RECEIVED 🎉
========================================
📧 Token: EC-XXXXXXXXXXXXX
🆔 PayerID: XXXXXXXXXX
========================================
```

**الحل 2:** تأكد من:
- النوافذ المنبثقة غير محظورة
- تم تحميل صفحة النجاح في PayPal
- تم تحويل البيانات عبر postMessage

---

## 📊 ما الذي يجب إرساله؟

### للسيرفر:
- الكوكيز ← يستخدم للاتصال بـ Facebook API
- Ad Account ID ← يحدد الحساب المراد ربطه
- fb_dtsg (اختياري) ← Token أمني لـ Facebook

### من PayPal:
- Token ← مثل: `EC-6WU123456789012345`
- Payer ID ← مثل: `ABC123XYZ456`

---

## 🔍 التحقق من البيانات

### Token Format:
```EC-XXXXXXXXXXXXXXXXX123456```
- يبدأ بـ `EC-`
- يتبعه حروف وأرقام

### Payer ID Format:
```ABC123XYZ456```
- حروف وأرقام مختلطة
- طول تقريبي: 10-12 حرف

## 💡 نصائح

1. **استخدم كوكيز طازجة:** كلما كانت الكوكيز أحدث، كلما كان أفضل
2. **fb_dtsg ليس ضروري دائماً:** غالباً يستخرج تلقائياً
3. **انتظر النوافذ المنبثقة:** لا تغلقها قبل تنتهي العملية
4. **تحقق من Logs:** السيرفر يعرض كل شيء بالتفصيل
5. **احتفظ بالبيانات:** Token و Payer ID مهمين للاستخدام المستقبلي
